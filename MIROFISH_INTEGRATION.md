## 为什么用 MiroFish？

### 做得到 vs 做不到

| 场景 | 纯 AI 分析 | + MiroFish 社会模拟 |
|------|:--:|:--:|
| 「喜欢天寒的读者多」 | ✅ | ✅ |
| 「如果天寒黑化，30% 的读者会弃书」 | ❌ | ✅ |
| 「评论区在分裂——两派人吵起来了」 | 能识别 | ✅ 能模拟冲突演化 |
| 「三个月后哪种走向订阅最高」 | ❌ | ✅ |
| 「新读者和老读者偏好差多远」 | ❌ | ✅ |

**一句话**：AI 分析告诉你「现在发生了什么」，MiroFish 模拟告诉你「如果这么写会发生什么」。

## 五步集成流程

### 第 1 步：评论采集 → AI 信号提取

```
输入：微信公众号评论 + B站评论 + 抖音评论
输出：偏好信号 JSON

{
  "character_sentiment": {
    "天寒": { "like": 0.82, "want_more_screentime": 0.65 },
    "肥鸭": { "like": 0.73, "want_more_screentime": 0.40 }
  },
  "plot_direction_requests": [
    { "theme": "天寒复仇", "support": 0.58 },
    { "theme": "游戏世界扩张", "support": 0.42 },
    { "theme": "现实线加深", "support": 0.35 }
  ],
  "tension_points": [
    "部分读者不希望天寒有感情线",
    "游戏 vs 现实比例有争议"
  ]
}
```

### 第 2 步：构建虚拟读者群体

基于真实评论的信号，在 MiroFish 中创建 100-1000 个 AI 智能体：

| 智能体属性 | 来源 |
|-----------|------|
| 角色偏好 | 评论情感分析 |
| 剧情口味 | 评论主题聚类 |
| 活跃度 | 评论频率 |
| 传播倾向 | 是否转发/推荐 |

### 第 3 步：多故事线并行模拟

注入 3 条候选故事线，让 MiroFish 模拟虚拟读者群体的反应：

```
故事线 A：天寒复仇线
├── 500 智能体模拟 4 周
├── 结果：留存率 82%，但「太黑暗」抱怨 15%
└── 传播指数：★★★☆

故事线 B：游戏世界扩张线
├── 500 智能体模拟 4 周
├── 结果：留存率 78%，新读者吸引力 +22%
└── 传播指数：★★★★

故事线 C：感情线展开
├── 500 智能体模拟 4 周
├── 结果：留存率 65%，老读者流失 18%
└── 传播指数：★★☆
```

### 第 4 步：创始人审核

**每条故事线必须通过以下审核才可入选：**

#### 🔴 法律红线（一票否决）

| 类别 | 零容忍内容 |
|------|-----------|
| **分裂国家** | 任何涉及分裂国家、破坏统一的内容 |
| **煽动颠覆** | 煽动颠覆国家政权、推翻社会主义制度 |
| **恐怖极端** | 宣扬恐怖主义、极端主义 |
| **民族宗教** | 煽动民族仇恨、民族歧视，破坏民族团结 |
| **色情淫秽** | 具体描写性行为、传播淫秽色情 |
| **暴力血腥** | 过度渲染暴力、血腥、恐怖，可能引起不适 |
| **未成年人** | 任何涉及未成年人不良导向的内容 |
| **虚假信息** | 编造传播虚假信息，扰乱社会秩序 |
| **侵犯权益** | 侵害他人名誉、隐私、知识产权 |

#### 🟡 伦理红线（需谨慎评估）

| 类别 | 约束 |
|------|------|
| **价值观导向** | 不得宣扬拜金主义、极端利己主义 |
| **歧视内容** | 不得包含性别、地域、职业、外貌等歧视 |
| **不良示范** | 自杀、自残、药物滥用等危险行为美化的描写需避免 |
| **历史虚无** | 不得歪曲、否定历史 |
| **社会公序** | 不得违背社会公序良俗 |
| **青少年保护** | 如有未成年读者，需格外注意导向 |

#### 🟢 创作一致性审核

| 维度 | 检查 |
|------|------|
| **世界规则一致** | 不违反 world_SOUL.md 中的任何规则 |
| **人设不崩溃** | 角色行为符合各自 SOUL.md 性格 |
| **伏笔合理** | 新展开不与未解伏笔冲突 |
| **遗忘机制** | 每章体现「记忆/遗忘」与「世界存续」的关系 |
| **艺术品质** | 故事张力、叙事节奏、情感层次 |

### 第 5 步：注入总控 → 生成新章

```
✅ 审核通过的故事线
        │
        ▼
  更新 story_framework.json
  ├── 调整 current_state
  ├── 标记新伏笔
  └── 注入选定的走向
        │
        ▼
  chapter_factory.py
  ├── 读取 story_framework.json
  ├── 引用角色 SOUL.md
  └── 生成 sequel_chapter_{N}.md
```
